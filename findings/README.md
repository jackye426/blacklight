# Findings

**Conclusions live here.** Everything under `findings/` is an **inference** — something we
reasoned to from the evidence in [`../investigations/`](../investigations). A finding is only
as good as the evidence it cites, so every finding links back to the observations that support
it.

- `architecture/` — how a system is structured, and why.
- `mechanisms/` — specific mechanisms explained (e.g. context compaction, permission layers).
- `comparisons/` — cross-system comparisons (e.g. Claude Code vs Codex vs Cursor).
- `reusable-patterns/` — design patterns extracted for reuse elsewhere.

Do not record an unsupported claim here. If the evidence isn't there yet, it's an open
question — put it in the relevant investigation's `unknowns.md`. See
[../RESEARCH-POLICY.md](../RESEARCH-POLICY.md).
